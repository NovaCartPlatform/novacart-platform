# Security Policy

## Overview

NovaCart Platform is committed to maintaining the security of the project and protecting its users, contributors, and infrastructure.

We appreciate responsible security research and encourage the community to report potential vulnerabilities privately.

---

# Supported Versions

The following versions are currently supported with security updates.

| Version | Supported |
|---------|-----------|
| 1.x | ✅ Yes |
| 0.x | ❌ No |

Only the latest stable release receives security updates.

---

# Reporting a Vulnerability

If you discover a security vulnerability, please do **not** create a public GitHub Issue.

Instead, report the vulnerability privately using the contact information below.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Screenshots (if applicable)
- Proof of Concept (if available)
- Suggested mitigation (optional)

---

# Response Timeline

NovaCart Platform aims to follow the response timeline below.

| Stage | Target Time |
|--------|-------------|
| Acknowledge the report | Within 48 hours |
| Initial investigation | Within 5 business days |
| Status updates | As required |
| Security fix | Based on severity |

These targets may vary depending on the complexity of the issue.

---

# Responsible Disclosure

Please allow adequate time for the vulnerability to be investigated and resolved before publicly disclosing any technical details.

Public disclosure before a fix is available may place users at unnecessary risk.

We appreciate coordinated and responsible disclosure practices.

---

# Scope

This policy applies to:

- Web Application
- REST APIs
- Authentication
- Infrastructure
- Kubernetes
- Docker
- CI/CD Pipelines
- Cloud Infrastructure
- Documentation

---

# Out of Scope

The following are generally considered out of scope:

- Typographical errors
- Feature requests
- Performance improvements
- Third-party service vulnerabilities
- Vulnerabilities requiring physical access

---

# Security Best Practices

Contributors should never commit:

- Passwords
- API Keys
- AWS Credentials
- Private Keys
- Certificates
- Secrets
- Sensitive customer information

Use environment variables and secure secret management instead.
