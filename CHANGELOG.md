## Versioning Strategy

NovaCart Platform follows Semantic Versioning.

Format:

Major.Minor.Patch

Example:

1.0.0

Where:

- Major = Breaking changes
- Minor = New features
- Patch = Bug fixes

---

## Release Types

### Major Release

Breaking architectural or functional changes.

Example:

2.0.0

---

### Minor Release

New features without breaking compatibility.

Example:

1.2.0

---

### Patch Release

Bug fixes and small improvements.

Example:

1.2.3
