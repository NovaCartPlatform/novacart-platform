# Code of Conduct

## Our Commitment

We are committed to providing a welcoming, respectful, inclusive, and professional environment for everyone participating in the NovaCart Platform project.

All contributors, maintainers, reviewers, and community members are expected to communicate professionally and treat others with respect.

## Expected Behavior

Examples of acceptable behavior include:

- Communicating respectfully and professionally.
- Providing constructive technical feedback.
- Respecting different opinions and levels of experience.
- Focusing discussions on project goals and technical outcomes.
- Accepting feedback professionally.
- Supporting a collaborative and inclusive environment.
- Following the project contribution and review guidelines.
- Taking responsibility for mistakes and working to correct them.

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, discrimination, or personal attacks.
- Insulting, threatening, or disrespectful comments.
- Publishing another person's private information without permission.
- Deliberately disrupting project discussions or workflows.
- Repeatedly ignoring contribution and review guidelines.
- Using inappropriate language or imagery.
- Any behavior that would be considered inappropriate in a professional workplace.

## Responsibilities

Project maintainers are responsible for:

- Clarifying acceptable standards of behavior.
- Reviewing reported violations.
- Taking appropriate and fair action.
- Protecting the privacy of anyone reporting an incident.
- Applying these guidelines consistently.

## Reporting

Concerns about behavior should be reported privately to the project maintainers.

Do not publish sensitive incident details in a public GitHub Issue, Pull Request, or Discussion.

Reports should include, when possible:

- A description of the incident.
- The date and location of the incident.
- Relevant links or screenshots.
- The names of involved participants.
- Any additional context that may help with the review.

All reports will be reviewed confidentially and handled as fairly and promptly as possible.

## Enforcement

Depending on the severity and frequency of the behavior, actions may include:

- A private warning.
- Removal of inappropriate comments or content.
- Temporary restriction from project participation.
- Rejection or closure of a Pull Request.
- Removal from the organization.
- Permanent removal from the project community.

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub repositories
- Pull Requests
- Issues
- Discussions
- Project documentation
- Jira work items
- Related communication channels
- Project meetings and reviews

## Professional Standards

All project participants are expected to:

- Keep technical discussions focused and constructive.
- Avoid unnecessary conflict.
- Respect code review decisions.
- Protect confidential information.
- Follow security and privacy guidelines.
- Contribute in good faith.
